# 최애 카메라 (Photo Frame Camera)

좋아하는 캐릭터 PNG를 카메라 화면에 띄워서, 마치 같이 있는 것처럼 사진을 찍는 PWA(웹앱)입니다.
첨부 레퍼런스(주술회전 howluv 카메라, 헌터×헌터 포토 프레임)에서 영감을 받아 개인용 프로토타입으로 제작되었습니다.

## ✨ 기능

- 후면/전면 카메라 전환
- 캐릭터 오버레이 — 드래그(이동), 두 손가락 핀치 또는 화면 우측 **+ / −** 버튼(크기), ↔ 좌우 반전, ↺ 초기화
- 셔터 → 카메라 영상 + 캐릭터를 합성한 단일 JPG 저장
- **앱 안에서 직접 이미지 추가** — 갤러리의 "+" 버튼으로 본인 이미지 등록. 브라우저 localStorage에만 저장되고 **서버에는 절대 업로드되지 않음**
- **이미지 자르기** — 미리보기 모달의 "✂ 자르기" 버튼으로 풀스크린 자르기 화면 진입. 사각형을 드래그로 이동, 모서리 핸들로 크기 조절, 3분할 가이드 라인 표시
- **PWA 지원** — 한 번 접속하면 홈 화면에 앱처럼 추가 가능, 오프라인에서도 동작
- 모바일 우선 디자인

## 📱 컴퓨터 없이 폰에서만 쓰기 (권장)

핵심: 무료 정적 호스팅에 **한 번만 올려두면**, 이후로는 폰 브라우저에서 URL만 열면 끝입니다. 그리고 한 번 접속한 다음 "홈 화면에 추가"하면 앱처럼 아이콘이 생기고 인터넷 없이도 켜집니다.

### 가장 빠른 방법 — Netlify Drop (1분, 회원가입 X)

1. 폰에서 [https://app.netlify.com/drop](https://app.netlify.com/drop) 접속
2. 페이지에 `ar_camera` 폴더(또는 그 안의 모든 파일)를 드롭/업로드
3. 발급된 `https://xxxxxxxx.netlify.app` URL을 폰에서 열기 → 카메라 권한 허용
4. (옵션) iOS Safari/Android Chrome 메뉴에서 **"홈 화면에 추가"** 선택 → 앱 아이콘 생성

> 주의: Netlify Drop은 **폴더 통째로** 올려야 경로가 맞습니다. 단일 파일들만 올리면 캐릭터/아이콘 경로가 깨질 수 있어요.
>
> 폰에서 폴더 업로드가 어려우면, 먼저 폴더를 zip으로 압축한 뒤 Netlify Drop에 zip을 그대로 드롭하면 자동으로 풀어서 배포해줍니다. (iOS '파일' 앱이나 Android에서 폴더 우클릭 → 압축)

### 대안 — GitHub Pages (영구 URL, 회원가입 필요)

1. 폰에 GitHub 모바일 앱 설치 + 회원가입
2. 새 public 저장소(repo) 만들기 (예: `choae-camera`)
3. 모든 파일을 저장소에 업로드 (모바일 앱의 "Add file" 기능)
4. 저장소 Settings → Pages → Source: `main` branch / `/ (root)` → Save
5. 1-2분 후 `https://[유저명].github.io/choae-camera/` 에서 접속

장점: URL이 바뀌지 않음, 영구 무료. 단점: 첫 설정이 Netlify Drop보다 번거로움.

### 대안 — Cloudflare Pages, Vercel

비슷한 방식. 회원가입 후 폴더 업로드 또는 GitHub 연동. URL은 둘 다 자동 HTTPS.

### 절대 안 되는 방법

- `file://` 로 폰에서 HTML을 직접 열기 → 카메라 API가 막혀서 안 됨
- HTTP(비보안) URL → 마찬가지로 카메라 권한 자체가 안 뜸
- 반드시 **HTTPS URL** 이어야 합니다

## 🖥️ 컴퓨터에서 로컬 테스트하고 싶을 때

```bash
cd ar_camera
python3 -m http.server 8000
```

브라우저에서 `http://localhost:8000` 접속. (localhost는 HTTPS 면제라 카메라 동작합니다.)

같은 와이파이의 폰에서 PC IP로 접속하려면 HTTPS가 필요한데, 가장 쉬운 건 ngrok:

```bash
# 한쪽 터미널
python3 -m http.server 8000
# 다른 터미널
ngrok http 8000
```

발급된 `https://xxxx.ngrok.app` 을 폰에서 열기.

## 🎨 캐릭터 이미지 추가 방법

### 방법 1) 앱 안에서 직접 추가 (권장 — 저작권 안전)

1. 카메라 화면 하단 갤러리에서 **"+"** 버튼 탭
2. 휴대폰 갤러리/사진 라이브러리에서 이미지 선택
3. **미리보기 모달**이 뜨고, 우하단 **"✂ 자르기"** 버튼으로 풀스크린 자르기 화면 진입 가능 (사각형 드래그 이동 / 4개 모서리 핸들로 크기 조절 / 3분할 가이드 라인). "적용" 누르면 잘린 결과로 모달 복귀
4. **추가** 버튼을 누르면 캐릭터 슬롯에 등록 → 바로 사용 가능
5. 추가된 캐릭터의 우상단 **"×"** 버튼으로 삭제

**이미지는 처리 없이 그대로 저장**됩니다. 자연스럽게 합성되려면 **배경이 투명한 PNG**(캐릭터만 잘라낸 cutout 이미지)를 추가하세요. 배경이 있는 일반 사진은 [remove.bg](https://www.remove.bg/), [Canva 배경 제거기](https://www.canva.com/features/background-remover/), Photoshop 등 외부 도구로 먼저 배경을 지운 PNG로 저장한 다음 추가하면 됩니다.

추가한 이미지는 **본인 브라우저에만 저장**됩니다 (localStorage 사용). 서버에 업로드되지 않으므로 저작권 보호 캐릭터를 본인 기기에서 사용해도 외부에 노출되지 않습니다. 저장된 캐릭터는 같은 브라우저로 다시 접속할 때 그대로 남아 있어요.

> 주의:
> - 같은 도메인이라도 **다른 브라우저/기기에서는 따로 추가**해야 합니다 (localStorage는 기기별로 분리됨).
> - 시크릿 모드에서는 세션이 끝나면 사라집니다.
> - 브라우저 데이터 삭제 시 함께 사라집니다.
> - 한 이미지는 추가될 때 가로/세로 최대 800px로 자동 리사이즈되어 저장됩니다 (저장 공간 절약).

### 방법 2) 폴더에 미리 PNG를 넣어두기 (배포 기본 캐릭터로)

`characters/` 폴더에 PNG를 추가하고 `app.js` 상단의 `DEFAULT_CHARACTERS` 배열을 수정:

```js
const DEFAULT_CHARACTERS = [
  { id: 'default_1', src: 'characters/my1.png', name: '슬롯 1', custom: false },
  { id: 'default_2', src: 'characters/my2.png', name: '슬롯 2', custom: false },
  // ...
];
```

이 방식은 호스팅 서버에 PNG가 같이 올라가므로, **저작권 보호 캐릭터는 절대 이렇게 넣지 마세요** (공개 URL 노출 시 문제). 본인이 직접 그린 오리지널이나 안전한 이미지에만 사용.

캐릭터를 추가/변경했다면 `service-worker.js`의 `ASSETS` 배열에 새 파일 경로를 추가하고 `CACHE_NAME`을 `'choae-camera-v3'` 처럼 올려주세요.

## ⚖️ 저작권 안내

이 앱 자체에는 어떤 캐릭터 이미지도 포함되어 있지 않습니다 (추상적인 도형 placeholder만 들어 있음). 사용자가 앱 안에서 추가한 이미지는 **본인 기기 브라우저에만 저장**되며 어떤 서버에도 업로드되지 않습니다.

따라서 본 앱은 다음과 같이 사용해도 안전합니다:
- 헌터×헌터, 주술회전 등 좋아하는 캐릭터 이미지를 폰에 다운받아 앱의 "+" 버튼으로 추가
- 결과 사진은 본인 폰에 저장
- URL을 친구에게 공유해도, 친구는 본인 placeholder만 보고 본인 이미지는 따로 추가해서 씀

만약 `characters/` 폴더에 캐릭터 PNG를 직접 넣어 호스팅하면 그건 **공개 배포**가 되므로 저작권 문제가 생길 수 있습니다. 이건 피하세요.

## 🔒 URL 접근 제어 (선택)

이 앱은 사용자 추가 이미지 방식이라 **공개 URL이어도 저작권 위험이 거의 없지만**, 그래도 본인만 / 가까운 사람들끼리만 쓰고 싶다면 다음 옵션이 있습니다.

### 약한 보호 (실제 보안은 아님)
- **추측 불가능한 URL**: Netlify Drop이 발급하는 `https://focused-bunny-a8f3c2.netlify.app` 같은 무작위 서브도메인은 사실상 검색에 안 잡히고 짐작도 어려워서, 링크를 아는 사람만 접근. 가장 손쉬운 방법.
- **로봇 차단**: 프로젝트 루트에 다음 내용으로 `robots.txt` 추가하면 검색엔진 인덱싱 방지.
  ```
  User-agent: *
  Disallow: /
  ```
- **클라이언트 비밀번호 게이트**: 앱 시작 시 간단한 패스워드를 묻는 화면 추가. 코드를 까보면 우회 가능하므로 보안용은 아니고, 모르는 사람의 우연한 접근만 차단.

### 강한 보호 (실제 인증)
- **Cloudflare Access** (Cloudflare Pages + Cloudflare Zero Trust 무료 플랜): 이메일 OTP, Google 로그인 등으로 진짜 인증 가능. 무료 50명까지.
- **Netlify Pro의 Password Protection** (유료, 월 $19~)
- **Vercel Password Protection** (Pro 플랜)

본인 + 친구 몇 명만 쓰는 정도라면 *추측 불가능한 URL + robots.txt* 조합이면 충분합니다. 필요하시면 비밀번호 게이트 코드도 붙여드릴게요.

## ⚠️ 알려진 제약

- iOS Safari는 PWA 모드에서 일부 카메라 옵션이 제한될 수 있습니다. 이상하면 일반 Safari 탭에서 열어보세요.
- 화면 회전 시 비디오 종횡비가 어긋나면 새로고침으로 해결.
- 캐릭터 교체 후 새로고침해도 안 바뀌면 캐시 때문 — `service-worker.js`의 `CACHE_NAME` 버전을 올려주세요.
